## What Went Well

1. The Case Folder became a real system of record.
   - `CaseFolderLayout`, manifest serialization, copied source metadata, reopen behavior, artifact discovery, and source-action audit all reinforce the same durable storage model.

2. The implementation respected ownership boundaries.
   - C# owns dock pane workflow, command gating, Case Folder operations, source copy, profile detection, source actions, and user-facing state.
   - Python and ArcPy stayed out of intake stories, which keeps Epic 2's processing boundary clean.

3. Review feedback improved the product, not just the code.
   - Story 1.3 review forced user-facing add-source behavior into the dock pane.
   - Story 1.4 review corrected TXT/CSV role inference and DWG-only classification.
   - Story 1.5 review caught stale state after reopen.
   - Story 1.6 review hardened non-blocking action behavior when audit state is corrupt or unwritable.

4. Testing grew alongside risk.
   - The C# console test runner expanded from scaffold checks into Case Folder, workflow, intake, reopen, and source-action coverage.
   - Contract validation and Python adapter tests continued to pass while intake behavior expanded.

Murat (Test Architect): "The most valuable test pattern is the filesystem-backed temporary Case Folder. It catches real recovery and path behavior without pretending ArcGIS Pro is available in unit tests."

## Challenges and Fixes

1. Scaffold correctness needed stronger validation.
   - Early review found a generic WPF-like scaffold was not enough. The fix moved the project toward ArcGIS Pro SDK-shaped entrypoints and made contract validation check that shape.

2. UI reachability needed repeated attention.
   - Story 1.3 showed that service behavior is insufficient if the dock pane cannot invoke it. Later stories kept commands and row state visible in the UI.

3. Intake classification had subtle source-role edges.
   - TXT/CSV and DWG-only cases exposed gaps in the detector's first pass. The final version handles deterministic TXT/CSV role inference and treats DWG-only as incomplete intake with missing roles.

4. Reopen created state hygiene pressure.
   - Creating a new case after a reopen originally retained prior source/profile/artifact state. The fix established a useful rule: successful new-case creation must reset all prior case-derived UI/session state.

5. Audit behavior needed to stay non-blocking.
   - Source open/reveal actions must not fail because the bounded audit artifact is damaged. The final audit append is best-effort for expected IO, JSON, and path failures.

Charlie (Senior Dev): "The pattern is clear: every story that touched persisted state needed one extra check for stale, corrupt, or missing data. That's not a flaw in the team; that's the nature of a Case Folder-driven workflow."

## Lessons Learned

1. Every user-facing acceptance criterion needs a dock pane path, not only a service method.

2. Case Folder paths must be treated as untrusted at action time.
   - Re-check existence and containment before opening, revealing, routing, or processing a file.

3. Non-blocking user actions include their bookkeeping.
   - If audit writing fails, the source action result should still be reported according to the action outcome.

4. Future workflow states should be acknowledged without being prematurely enabled.
   - Reopen can report unsupported future states without inventing Epic 2-6 behavior.

5. Contract validation is valuable when it checks behaviorally important shape.
   - It should continue validating stable artifact names, schema fields, adapter boundaries, and expected source files as new epics add artifacts.

Epic 2 should build directly on Epic 1:
