# Target Computer Deployment Tools

This folder stages the support files needed for ArcGIS Pro test deployment.

## What To Copy

Copy the whole `deployment/target-computer-tools` folder to the target computer.

The staged `package/` folder is generated by:

```powershell
.\tools\stage_target_deployment.ps1 -Configuration Release
```

It contains:

- `ParcelWorkflowAddIn.esriAddInX`
- `ProcessingTools/`
- `Contracts/`
- `deployment_manifest.json`

## Install On The Target Computer

From the copied folder on the target machine:

```powershell
.\scripts\install_target_tools.ps1
```

Default install root:

```text
C:\Sidwell\ParcelWorkflow
```

The installer copies the Python processing tools and contracts, creates a target-configured add-in package, and can launch the add-in installer.

## Python Environment

Manually copy the `arcgispro-survey-ai` folder to:

```text
C:\Sidwell\ParcelWorkflow\python-env
```

The installer will automatically use:

```text
C:\Sidwell\ParcelWorkflow\python-env\python.exe
```

If the target machine already has a working ArcGIS Pro Python clone, use it instead:

```powershell
.\scripts\install_target_tools.ps1 -PythonExe "C:\Path\To\Existing\ArcGISPython\python.exe"
```

If you intentionally want to bundle the Python environment into this package, regenerate it with:

```powershell
.\tools\stage_target_deployment.ps1 -Configuration Release -IncludePythonEnv
```
